Chapter 6: Programming Languages
Term Definition
Machine language Sequences of instructions that are encoded as numeric digits.
Debugging A process in which errors are located and corrected.
Assembler Programs that convert mnemonic expressions into machine language instructions.
Assembly language A mnemonic system for representing programs..
Translator An application that translates programs expressed in high-level primitives into machine-language programs.
Compiler A program that compiles several machine instructions into short sequences.
Interpreter An alternative to translators, that executed instructions as they are tanslated rather than just recording them.
Cross-Platform Software Software that is independent of an operating system's design as well as the machine's hardware design and is therefore excutable throughout a network.
Natural language Spoken languages like English, German, and Latin that are precisely defined by grammar rules.
Formal language Programming languages that evolved over time without foraml grammatical rules.
Machine independence A program that could theoretically be used on any machine by applying the appropriate compiler.
High-level Programming language A programming language that provides more abstractions and makes it easier for people to read and write a program.
Object-oriented programming (OOP) A computer programming model that organizes software design around data, or objects, rather than functions and logic.
Scripting language Programming languages that are typically used to preform administratvie tasks rather than to develop complex programs.
Date types: Integer
  • Refers to numeric data consisting of whole numbers
Float
  • Refers to numeric data that might contain values other than whole numbers.
String
  • A sequence of characters and the most commonly used data type to store text.
Primitive data type Data types in programming languages such as int for integer and char for character.
Data structure The conceptual shape or arrangement of data.
Python collection data structures: List
  • Used to store multiple items in a single variable (created using square brackets)
Tuple
  • A collection which is ordered and unchageable (created using round brackets).
Set
  • A collection which is unordered, unchangable, and unindexed (created using curly brackets)
Dictionary
  • A collection which is ordered, changable, and does not allow duplicates.
Function A set of instructions for performing a task that can be used as an abstract tool by other program units.
Local variable A variable that can be referenced only within that function.
Global variable A variable whose scope is not restricted to a particular part of a program.
Source program A program in its orginal form.
Object program A translated version of a program.
